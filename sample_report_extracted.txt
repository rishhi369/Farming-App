Abstract 
 
In today's rapidly urbanizing world, civic issues such as damaged roads, overflowing garbage, 
pollution spots, and safety hazards go unreported for extended periods due to a lack of 
accessible, real-time reporting mechanisms between citizens and municipal authorities. 
Existing systems are fragmented, unresponsive, and fail to create a sense of community 
ownership. 
This project presents CivicTrack, a cross-platform mobile application built using React 
Native (Expo) that empowers citizens to report local civic issues directly to relevant 
authorities, receive crisis alerts, participate in community announcements, and engage with 
community forums  all from their smartphone. 
CivicTrack bridges the communication gap between communities and government bodies by 
making civic reporting simple, real-time, and accessible. Citizens can submit issue reports 
with photos, GPS coordinates, and descriptions; receive real-time push notifications about local 
crises; and interact with their community through an organized announcement system. 
Future iterations aim to include role-based access for authorities, admin dashboards with issue 
heatmaps, and gamification rewards for active citizens.

1. Introduction  
 
Overview of the project  
CivicTrack is a civic engagement mobile application designed to bridge the gap between 
citizens and local authorities. It allows residents to report local problems, stay informed about 
community events and crises, and participate in area-specific community forums  all in one 
place. 
Brief Summary 
  The application provides: 
 Issue Reporting :- Citizens can report civic problems (roads, garbage, water, 
electricity) with image attachments and live GPS location. 
 Community Forum :- Users can join location-based communities, view 
announcements, and interact with community members. 
 Push Notifications :- Real-time crisis alerts and emergency broadcasts are delivered 
via Firebase Cloud Messaging. 
 User Profiles :-  Personalized profiles to track submitted applications and community 
memberships.

2. Problem Statement 
Definition of the Problem  
Civic issues in urban and semi-urban areas often go unaddressed for weeks or months 
due to a lack of direct communication channels between citizens and local authorities. 
Citizens have no standardized platform to: 
 Report problems with evidence (photos + location) 
 Track the status of their reported issues 
 Receive timely alerts about local crises 
 Engage with their community on civic matters 
Objectives  
 Enable citizens to submit detailed issue reports with images, GPS location, and 
descriptions. 
 Provide real-time push notifications for local crisis alerts and community 
broadcasts. 
 Build a community forum ecosystem where locality-specific announcements can 
be posted and viewed. 
 Allow users to track their applications and view submission history. 
 Support a role-based flow distinguishing regular citizens from community 
admins. 
Outcomes  
 For Users: Faster, easier, and transparent civic issue reporting from any smartphone. 
 
 For Communities: Real-time information sharing and announcements within 
locality-based groups. 
 
 
 Technical: Scalable, real-time architecture using Firebase Firestore capable of 
handling concurrent users and live data updates.

6. User Manual
 
 
Getting Started: 
1. Download the App — Via Expo Go (development) or the distributed APK. 
2. Create an Account — Tap Sign Up, enter your name, email, and password. 
3. Log In — Use your registered email and password. 
 
Reporting a Civic Issue: 
1. From the Home Screen, tap "Apply / Report Issue". 
2. Fill in the issue details: 
 Select issue category (Road, Sanitation, Water, Electricity, etc.) 
 Add a description of the problem 
 Attach a photo (camera or gallery) 
 Tap "Use My Location" to auto-fill GPS coordinates 
3. Tap Submit to send the report. 
 
Joining a Community 
1. Navigate to the Community tab. 
2. Browse available communities (by area or name). 
3. Tap a community → Tap "Join". 
4. Once joined, view Announcements and Members in that community. 
 
Viewing Announcements 
1. Go to Community → Select your community. 
2. Tap "Announcements" to view posts from admins. 
3. Enable notifications to receive real-time alerts. 
 
Notifications 
 Crisis Alerts are sent automatically by community admins via push notifications. 
 Ensure notifications are enabled in your phone settings for CivicTrack.

7. Conclusion 
 
Summary of Findings  
     Effective Civic Reporting: 
 Citizens can report issues in under 2 minutes with image and GPS attachment. 
 Community-specific reporting ensures issues reach the right authority group. 
Community Engagement: 
 Announcement system enables real-time community-wide communication. 
 Community forum fosters local ownership and peer collaboration. 
Technical Performance: 
 Firebase Firestore provides real-time data sync with sub-second latency. 
 Scalable architecture handles growing communities with concurrent users. 
 Cloudinary ensures efficient image storage and CDN-level delivery. 
 
 
       Recap of Results  
 Functional Deliverables 
o Real-Time Chat System 
o End-to-end encrypted messaging with Socket.io 
o Supports 1:1 and group conversations 
 Sentiment Analysis Engine 
o Hybrid model (Gemini API + Sentiment Library) 
o Accuracy: 89% (English), 82% (Hinglish) 
  User Dashboard 
o Visualizes sentiment trends (weekly/monthly) 
o Emotion breakdown pie charts
